# データ量子化とメトリクス駆動マージ — 2026-03-05

## 背景

engram の競合分析（Mem0/OpenMemory, mcp-memory-service, Memora）から出発し、
知識ノードのライフサイクルに関する新しい抽象モデルに到達した。

mcp-memory-service は知識グラフのエッジ（causes / fixes / contradicts）を
自動推論エンジンで生成しようとしたが、精度が低く（Issue #546）、
エージェントがグラフを辿る API も存在しない。
根本的な問題は **関係性をノード側に埋め込んだこと** にある。

---

## データ量子化 — 抽象モデル

### 核心: 観測者の状態が結果を変える

```
状態ベクトル S × 射影行列 P → 可能性分布 D

S: 観測対象の現在状態（feelings, query, 何でもいい）
P: 観点の集合（各行が1つの「見方」）
D: 各観点での応答強度
```

phi-agent での実証:
- S = feelings [4D]
- P = modeWeights（5行×4列、種族ごとに異なる）
- D = 各 mode のスコア → argmax で1つ選ぶ

同じ feelings でも moth は `hot` を選び、scholar は `deep` を選ぶ。
ノード（Sphere のセル、食料、磁場）には何の細工もない。変わるのは常に観測者側。

### 3つの性質

| 性質 | 意味 | phi-agent での対応 |
|---|---|---|
| **重ね合わせ** | 1ノードが複数の射影に非ゼロで応答する | 同じ feelings が hot にも explore にも非ゼロスコアを返す |
| **観測による収縮** | argmax や top-N で1つに決定する | winner = argmax(score) |
| **配分の不在 = 不可視** | P に行がなければその可能性は存在しない | modeWeights に `deep` がない moth は deep を選べない |

### P を誰が持つか

```
ノード側に持たせる → 知識グラフ（関係性の固定化、mcp-memory-service の失敗）
観測者側に持つ    → 可能性クエリ（phi-agent のパターン）
環境が持つ        → 磁場（Sphere の globalField）
```

**量子化されていないデータとは、P が単位行列であるデータ。**
1つの角度でしか照らせない。量子化とは P に複数行を持たせること。

### 重要な区別: 観測者の設計であってノードの生物化ではない

ノードにメトリクスがあれば状態を表現しており、反応はする。
だが **ノード間の連結は観測者の状態が繋ぐ**。
ノード自体が生物化するのではなく、観測者の射影行列 P が
「どのノードとどのノードが関連するか」を決定する。

スフィアノードのようにメトリクスがあれば、ノードの状態が射影の入力となり、
アクセス対象を能動的に変化させることはできる。
だがそれはノードが自律的に連結するのではなく、
行動パターン（modeWeights の行）が事前定義されていて初めて成立する。

行動パターンの定義数 = 生物の複雑さ。射影行列の行数が可能性の幅を決める。

## メトリクス駆動マージ

### 動機

mcp-memory-service の consolidation が機能しなかったのは、
**外部のシステムが「似てるからマージしろ」と決めていた** から。
ノード自身のメトリクスが駆動するなら話が根本的に違う。

### 概念 1: 遺言マージ（死ぬ前の引き継ぎ）

```
遺言マージ:
  ノードA: weight 低い、ttl 残り少ない、でも hitCount がある
  ノードB: 同近傍、weight 高い、新しい
  → A が「自分はもう死ぬが、この知識は B に引き継げ」
```

ノードが叫ぶというのは比喩ではなく、メトリクスの組み合わせが
特定の状態に入ったときに Digestor が検知できるということ。

### 概念 2: メニアルマージ（ゴミ同士の合体）

```
メニアルマージ:
  ノードA: weight 低い、もうすぐ死ぬ、単体ではゴミ
  ノードB: weight 低い、もうすぐ死ぬ、単体ではゴミ
  どちらも近傍にいる = 意味が重なっている
  → 合体して1ノードに。断片が補完し合い、少し価値が出る
  → 2つのゴミが1つのまともなノードになる
  → データプールからゴミが1つ減る（清掃とサルベージの同時実行）
```

**TTL は合算する。** 2つの残り時間を足して合体ノードに与える。
これが合体のインセンティブ — 単体では死ぬ運命だったが、
TTL ボーナスで猶予が伸び、次の recall でヒットするチャンスを得る。
使われれば生き延びる。使われなければ予定通り死ぬ。
「最後のチャンス」であって「救済」ではない。代謝の判定は変わらない。

### Digestor の第3の行動

現在の Digestor は `promote / expire` の2択。
3つ目の行動として **merge** が自然に生える:

| 行動 | 条件 | 結果 |
|------|------|------|
| **expire** | ttl <= 0 かつ weight 低い | 死 |
| **promote** | weight >= 3 かつ hitCount >= 5 | fixed（永続化） |
| **merge（遺言）** | ttl 残少 かつ hitCount > 0 かつ近傍に weight の高いノードがある | 知識を引き継いで死ぬ |
| **merge（メニアル）** | 双方 weight 低い かつ ttl 残少 かつ cosine_similarity >= 閾値 | 合体、TTL 合算、content 合成 |

これは代謝モデルの自然な拡張であって、
外部が機械的に統合するのとは根本的に動機が違う。
ノードの状態が「マージすべきだ」と示しているから行う。

### ベクトル DB との親和性

セマンティックベクトル DB を使っている以上、
近傍ノード（= マージ対象候補）は常に自然と存在する。
特別なグラフ構造を張る必要がない。

## 生物ノード — 確率的自己決定と共鳴

### バッチ処理 vs 確率的自己決定

```
フィルタリング（開発者の都合）:
  Digestor が全ノードを走査 → 条件に合うものを処理
  → 条件を満たす全員が同じ行動を取る
  → 創発は起きない。ただのバッチ処理

確率的自己決定（ノードの都合）:
  各ノードが自分のメトリクスから行動確率を算出
  → 同じ状態でも do するノードと no するノードがいる
  → 共鳴シグナルの合計が確率を押し上げる
  → 創発が起こり得る
```

phi-agent の `shouldReturn` が実証:
```
prob = clamp((desire - 0.5) × 2)
return = random() < prob
// 同じ感情でも帰る奴と帰らない奴がいる
```

### ノードの行動確率

```
node.assess():
  mergeUrge = f(weight, ttl, hitCount)  // 0.0 〜 1.0
  resonance = Σ(共鳴シグナル)            // 周囲から受け取った圧
  prob = clamp(mergeUrge + resonance)

  if random() < prob → do merge
  else               → no（今回は見送り、次の tick で再判定）
```

同じ状態の100ノードのうち70が merge して30が生き残る。
その30が次の tick でまた判定する。**時間方向の分岐** が生まれる。
早く merge したノードと粘ったノードで結果が変わる。
粘った方が後から来た新ノードと merge してより良い結果を得るかもしれない。
あるいはそのまま死ぬかもしれない。これは設計できない。創発するしかない。

### 共鳴モデル — リンクなき遠方通信

エージェントがいない世界ではノード間にパスがない。
リンクを張るには「どういう意味で」を誰かが判断する必要があり、
それは量子化の原則（関係性を固定しない）と矛盾する。

共鳴はこれを解決する:

```
resonance(action, nodeA) = Σ over all nodes B:
  compatibility(A, B)           // 状態の類似度（双方 weight 低い、ttl 少ない等）
  × decay(distance(A, B))      // cosine distance による減衰
  × actionAffinity(B, action)  // B がその行動に対して持つ親和度

if resonance > threshold → action の発動確率が上がる
```

- 近傍のゴミ同士 → 共鳴強、距離減衰なし → メニアルマージ高確率
- 遠方だが状態が酷似 → 共鳴中、距離減衰あり → 条件次第で発動
- 遠方で状態もバラバラ → 共鳴ゼロ → 何も起きない

**リンクを張らずに遠方通信を実現している。**
共鳴は一時的な現象であって永続的な関係ではない。
次の tick では状態が変わり、共鳴相手も変わる。
量子化の原則「関係性を固定しない」と整合する。

### 生物ノードのメソッド群

```
// ---- 生存系 ----
survive()       // decay に耐え次の tick へ
feed()          // recall ヒット時。weight += 0.35, hitCount++
starve()        // recall されない tick が続く

// ---- 成長系 ----
promote()       // weight >= 3 && hitCount >= 5 → fixed
split()         // hitCount 異常高 + 参照角度の分散大 → 分裂（条件未確定）

// ---- 社会系 ----
menialMerge()   // 近傍のゴミノードと合体。TTL 合算
bequeath()      // 遺言マージ。死ぬ前に強いノードに引き継ぐ
repel()         // 近傍に意味が近すぎる → embedding 微調整で棲み分け
signal()        // 特定状態で共鳴シグナルを発する

// ---- 死亡系 ----
expire()        // ttl <= 0 && weight 低い → 自然死
decay()         // 毎 tick の weight 減少 → 老化
flagged()       // agent から engram_flag → 急速 weight 低下 → 病死

// ---- 環境応答系 ----
hibernate()     // project idle → ttl 停止 → 冬眠
wake()          // activity 復帰 → 代謝再開

// ---- 感覚系 ----
sense()         // 近傍 N 件を認識（merge/repel の前提）
assess()        // メトリクスから行動確率を算出（computeFeelings 相当）
```

`assess()` が全行動の起点。phi-agent の `computeFeelings` と同構造。
